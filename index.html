WHY NEON POSTGRES?

Neon provides a persistent, reliable online database for the leaderboard. With Render's free tier, SQLite files can be wiped when the service sleeps or restarts—causing scores to disappear. Neon keeps data safely stored with a secure connection string, so the leaderboard updates instantly for every player and never loses scores.

💾 NEON POSTGRES

PERSISTENT STORAGE

CLOUD ARCHITECTURE

Persistent Data Storage

Neon PostgreSQL provides secure, reliable database infrastructure for ESCAPE THE WASK's leaderboard, player scores, and game data persistence.

Database Architecture

  • PostgreSQL 15+ relational database
  • Multi-table schema design
  • Indexed queries for performance
  • Transaction support for data integrity
  • JSON data type support

Data Tables

  • players: User accounts & profiles
  • leaderboard: Global scores & rankings
  • game_sessions: Player game history
  • boss_defeats: Boss challenge records
  • achievements: Player accomplishments

Security Features

  • SSL/TLS encrypted connections
  • Connection pooling (PgBouncer)
  • Role-based access control
  • SQL injection prevention (parameterized queries)
  • Automated daily backups

Performance Optimization

  • Database indexing on frequently queried columns
  • Composite indexes for complex queries
  • Query optimization & EXPLAIN analysis
  • Connection pooling for reduced latency
  • Automated VACUUM & ANALYZE

Why Neon Postgres? Unlike SQLite which loses data on server restarts, Neon Postgres provides serverless, auto-scaling database infrastructure with guaranteed data persistence. This ensures player scores and leaderboard data are never lost, even during cloud infrastructure updates.
